┌──────────────────────────────────────────────────────────────────────────┐
│ WHAT IS DRY RUN?                                                        │
└──────────────────────────────────────────────────────────────────────────┘

  A simulation of the backup process that shows EXACTLY what would happen
  WITHOUT actually copying, modifying, or deleting ANY files.

  Think of it as a "backup preview" or "what-if analysis"


┌──────────────────────────────────────────────────────────────────────────┐
│ THE MAGIC COMMAND                                                        │
└──────────────────────────────────────────────────────────────────────────┘

  rsync -aHv --dry-run --itemize-changes --stats /source/ /destination/
         │             │                   │
         │             │                   └─── Show summary statistics
         │             └────────────────────── Show file-by-file changes
         └──────────────────────────────────── DON'T ACTUALLY DO ANYTHING!


┌──────────────────────────────────────────────────────────────────────────┐
│ HOW IT WORKS                                                             │
└──────────────────────────────────────────────────────────────────────────┘

  Step 1: Rsync scans source and destination directories
          ↓
  Step 2: Compares files (size, timestamp, checksum)
          ↓
  Step 3: Determines what WOULD change
          ↓
  Step 4: Outputs detailed report
          ↓
  Step 5: STOPS (doesn't actually make changes)


┌──────────────────────────────────────────────────────────────────────────┐
│ OUTPUT INTERPRETATION                                                    │
└──────────────────────────────────────────────────────────────────────────┘

  Symbol         Meaning                  Action
  ───────────────────────────────────────────────────────────────────────
  >f+++++++++    New file                 Would CREATE
  >f.st......    Modified file            Would UPDATE
  *deleting      Deleted file             Would REMOVE
  .f.........    Unchanged file           Would SKIP

│ SAMPLE OUTPUT                                                            │
└──────────────────────────────────────────────────────────────────────────┘

  .f......... photo.jpg                ← File unchanged (SKIP)

  Number of files: 1,247

┌──────────────────────────────────────────────────────────────────────────┐
│ TECHNICAL IMPLEMENTATION                                                 │
└──────────────────────────────────────────────────────────────────────────┘

  Your Application (Python)
        ↓
  RsyncEngine.run_rsync(dry_run=True)
        ↓
  Execute: rsync --dry-run --itemize-changes
        ↓
  Parse output → _parse_itemize_changes()
        ↓
  Categorize files:
    • new_files[]
    • updated_files[]
    • deleted_files[]
    • unchanged_files[]
        ↓
  Send to GUI via progress_callback
        ↓
  Display formatted report


┌──────────────────────────────────────────────────────────────────────────┐
│ VERIFICATION PROCESS                                                     │
└──────────────────────────────────────────────────────────────────────────┘

  BEFORE dry run:
    $ ls -la /destination/
    → Note: 100 files, 5.2 GB

  DURING dry run:
    $ rsync --dry-run ... /source/ /destination/
    → Shows: Would add 5 files, update 3 files

  AFTER dry run:
    $ ls -la /destination/
    → Still: 100 files, 5.2 GB (UNCHANGED)

  ✅ Confirmed: Dry run made NO changes

┌──────────────────────────────────────────────────────────────────────────┐
│ COMMON SCENARIOS                                                         │
└──────────────────────────────────────────────────────────────────────────┘

  Scenario 1: First Backup
  ─────────────────────────────────────────────────────────────────────
  Dry run shows: All files as "new" (>f+++)
  Expected: ✅ Correct if destination is empty
  Action: Proceed with backup

  Scenario 2: Incremental Backup (no changes)
  ─────────────────────────────────────────────────────────────────────
  Dry run shows: All files as "unchanged" (.f...)
  Expected: ✅ Correct if nothing changed
  Action: Can skip backup (nothing to do)

  Scenario 3: Incremental Backup (with changes)
  ─────────────────────────────────────────────────────────────────────
  Dry run shows: Mix of new, updated, unchanged
  Expected: ✅ Normal for active data
  Action: Review changes, then proceed

  Scenario 4: Unexpected Deletions
  ─────────────────────────────────────────────────────────────────────
  Dry run shows: Many "*deleting" entries
  Expected: ⚠️ Warning! Investigate before proceeding
  Action: Check source path is correct
